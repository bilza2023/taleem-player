One-Way Data Flow

Taleem enforces a strict one-way flow of information.

Nothing flows upward.

The Direction

Schema → Renderer → Engine → DOM

No Upward Communication

Lower layers do not send information back upward.

  • engines do not inspect slide data
  • renderer does not know how slides are used
  • schema does not react to runtime behavior

Why This Is Strict

Two-way systems accumulate hidden dependencies.

One-way systems remain legible.

You can reason about behavior by reading from top to bottom.

What Happens If This Is Broken

When data flows upward:

  • engines start parsing content
  • UI becomes coupled to structure
  • features become impossible to remove

The system becomes fragile.

The Benefit

One-way flow allows:

  • multiple engines
  • multiple UIs
  • future extensions

Without rewriting content.

This is what makes Taleem durable.
