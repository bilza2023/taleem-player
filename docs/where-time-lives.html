Where Time Lives

Time exists in exactly one place in the Taleem system.

Time lives only in the player.

Time Is Not Content

Time is not part of a lesson.

A lesson describes what is shown, not when it is shown.

Treating time as content makes lessons fragile and impossible to reuse.

The Only Time-Aware Layer

The time-based engine (taleem-player) is the only layer allowed to reason in seconds.

It may:

  • schedule slide visibility
  • advance slides automatically
  • control progressive reveal

Where Time Must Not Exist

Time must not appear in:

  • schema definitions
  • renderers
  • browser engines
  • UI components

If time appears in any of these places, the boundary has been violated.

Why This Rule Is Strict

When time leaks into other layers:

  • content becomes tied to a single playback mode
  • reuse becomes impossible
  • systems become tightly coupled

By isolating time, Taleem keeps lessons flexible without modifying the content itself.

The Consequence

A single deck can be:

  • browsed manually
  • played automatically
  • synchronized with audio
  • revealed progressively

All without rewriting the lesson.

The Rule

If a feature needs time, it belongs in the player.

If it does not, it does not.
