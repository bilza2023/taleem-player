taleem-player

taleem-player is the time-based engine in the Taleem system.

It is the only part of the system that understands time.

What the Player Does

taleem-player:

  • controls slide visibility using time
  • advances slides automatically
  • coordinates progressive reveal

Time is treated as a control signal, not as content.

Where Time Lives

Time exists only in the player.

No other layer may:

  • store time values
  • schedule transitions
  • reason in seconds

This rule is strict.

Player-Ready Slides

taleem-player assumes slides are already rendered.

It does not validate schema. It does not render templates.

If a slide deck is malformed, the player is allowed to fail loudly.

Progressive Reveal

The player may reveal parts of a slide over time.

This behavior is driven entirely by timing data, not by slide logic.

Typical Uses

The player engine is suited for:

  • auto-running presentations
  • slides synchronized with audio
  • guided lessons
  • recorded explanations

What the Player Does Not Do

taleem-player does not:

  • render HTML
  • inspect slide structure
  • modify templates
  • own UI controls

The Boundary

taleem-player answers one question:

“What should be visible at this moment in time?”

If a feature does not depend on time, it does not belong here.
