Demos as Executable Docs

In Taleem, demos are not examples.

They are executable documentation.

The Contract

A demo represents the expected behavior of the system at a given point in time.

If the demo breaks, the system is considered broken.

Why This Matters

Written documentation can drift.

Executable demos cannot.

They either run correctly or they do not.

What Demos Must Not Do

Demos must not:

  • contain hidden logic
  • work around system flaws
  • mask errors

A demo should reflect the system, not compensate for it.

Relationship to the Gold Deck

Most demos are built on top of the Gold Deck.

This ensures complete coverage and early detection of regressions.

The Rule

If documentation and demos disagree, trust the demo.

Then update the documentation.
