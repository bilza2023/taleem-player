The Gold Deck

The Gold Deck is the reference implementation of Schema v1.

It is not an example. It is a contract.

What the Gold Deck Is

The Gold Deck contains one instance of every supported slide type.

Each slide is populated with minimal, representative data.

Its purpose is coverage, not presentation.

Why It Exists

The Gold Deck answers one question:

“Does the system still work?”

Any change to the schema, renderer, or engine must continue to render the Gold Deck correctly.

The Stability Rule

If the Gold Deck breaks, the system is broken.

There are no exceptions.

Fix the system, not the deck.

What the Gold Deck Is Not

The Gold Deck is not:

  • a demo
  • a tutorial
  • a design showcase
  • editable content

It is intentionally plain. Visual polish is irrelevant here.

How It Is Used

The Gold Deck is used to:

  • validate schema changes
  • test renderers
  • verify engines
  • detect regressions

If all slides in the Gold Deck render correctly, the system is considered stable.

One Deck, One Truth

There is only one Gold Deck per schema version.

It evolves only when the schema evolves.

This keeps the system honest.
