How to Run the Demo

Taleem demos are intentionally simple.

They are designed to remove as many variables as possible.

What the Demo Does

The demo:

  • loads a validated deck
  • renders all slides
  • displays them using an engine

There is no application logic beyond this.

What You Should Look For

When running the demo, focus on:

  • whether slides render correctly
  • whether navigation behaves as expected
  • whether playback is stable

Visual styling is not important here.

Failure Is Informative

If the demo fails to run:

  • the schema may be invalid
  • a renderer may be broken
  • an engine boundary may have been violated

Fix the system. Do not work around the demo.

Why Demos Are Minimal

Minimal demos are easier to reason about.

They fail loudly and early, which is exactly what is needed.
