Schema v1

Schema v1 defines what a Taleem deck is.

It is a structural contract, not an implementation detail.

What the Schema Does

The schema defines the shape of slide data.

It specifies:

  • which slide types exist
  • what data each slide type accepts
  • how slides are ordered

The schema does not define how slides look, how they animate, or how they are played.

Pure Data Only

Schema v1 contains only data.

It has:

  • no rendering logic
  • no layout rules
  • no UI concepts
  • no playback behavior

This separation allows the same deck to be rendered, browsed, or played without modification.

Slide Types Are Explicit

Every slide declares its type explicitly.

Each type accepts a fixed set of data fields. Unknown or malformed data is rejected.

This keeps slide behavior predictable and prevents silent layout drift.

Validation Is Canonical

Schema v1 is validated using a strict schema definition.

If a deck does not validate, it is not a Taleem deck.

There are no partial modes, no best-effort rendering, and no fallback behavior.

Why This Matters

By treating content as structured data:

  • content can be generated programmatically
  • rendering stays deterministic
  • systems remain testable

Schema v1 is the foundation that makes the rest of the system stable.
